Elementos de Páginas Web y Diseño

Tema 1: Elementos de Páginas Web

I. Orígenes de la Web

  • Web 2.0: La Web 2.0 se refiere a una evolución de la web centrada en la participación del usuario, la colaboración y el intercambio de información. Se caracteriza por plataformas de redes sociales, wikis y otras herramientas colaborativas.
  • Web 3.0: También conocida como la web semántica, la Web 3.0 busca hacer que la web sea más inteligente mediante la interpretación y conexión de datos a través de metadatos y ontologías. Su objetivo es mejorar la interacción entre los usuarios y la información.
  • La W3C y los estándares: La World Wide Web Consortium (W3C) es una organización que desarrolla estándares para asegurar la interoperabilidad y el buen funcionamiento de la web. Su labor incluye la creación de recomendaciones y estándares como HTML, CSS y XML.

II. Lenguajes de etiquetas

  • SGML: Standard Generalized Markup Language es un estándar que define un sistema de marcado para documentos. SGML fue la base para la creación de XML y HTML.
  • XML: Extensible Markup Language es un lenguaje de marcado que define un conjunto de reglas para codificar documentos en un formato que es legible tanto para humanos como para máquinas. XML es utilizado para el intercambio de datos entre sistemas.
  • HTML: HyperText Markup Language es el lenguaje estándar para la creación de páginas web. Define la estructura del contenido mediante etiquetas.
  • XHTML: Extensible HyperText Markup Language es una versión más rigurosa de HTML que sigue las reglas de XML. Combina la flexibilidad de HTML con la estructura estricta de XML.
  • HTML5: Es la versión más reciente de HTML. Incluye nuevas funcionalidades y elementos que mejoran la capacidad de las páginas web para manejar multimedia y gráficos, así como soporte para aplicaciones web más avanzadas.

III. Otros lenguajes relacionados al desarrollo web

  • CSS/CSS3: Cascading Style Sheets es un lenguaje utilizado para describir la presentación de un documento escrito en HTML o XML. CSS3 es la última versión y agrega características avanzadas como animaciones y diseños adaptativos.
  • JavaScript: Es un lenguaje de programación que se utiliza para crear interactividad en las páginas web. Permite realizar cambios dinámicos en el contenido y el diseño sin necesidad de recargar la página.
  • Ajax: Asynchronous JavaScript and XML es una técnica que permite actualizar partes de una página web de manera asíncrona, sin recargar toda la página. Mejora la experiencia del usuario mediante la carga dinámica de contenido.
  • Tecnologías y lenguajes del lado del servidor: Incluyen lenguajes como PHP, Python, Ruby y frameworks como Node.js, que se ejecutan en el servidor para gestionar la lógica de negocio, interactuar con bases de datos y generar contenido dinámico.
  • Navegadores: Son aplicaciones que permiten visualizar y navegar por páginas web. Ejemplos incluyen Google Chrome, Mozilla Firefox, Safari y Microsoft Edge.
  • Motores renderizados: Son los componentes dentro de los navegadores que interpretan el código HTML, CSS y JavaScript para mostrar las páginas web. Ejemplos son Blink (utilizado en Chrome) y Gecko (utilizado en Firefox).

IV. Herramientas de desarrollo

  • Software WYSIWYG: Son herramientas que permiten diseñar páginas web mediante una interfaz gráfica, mostrando cómo se verá el resultado final sin necesidad de escribir código. Ejemplos incluyen Adobe Dreamweaver y Microsoft Expression Web.
  • Otras alternativas:
    • Figma: Herramienta de diseño colaborativo para interfaces que permite crear prototipos y diseños de alta calidad.
    • InVision: Plataforma para diseñar prototipos interactivos y colaborar en el proceso de diseño.
    • Zeplin: Herramienta que facilita la transferencia de diseños entre diseñadores y desarrolladores, generando especificaciones y activos.
    • GitHub: Plataforma de desarrollo colaborativo que utiliza Git para el control de versiones, facilitando el trabajo en equipo en proyectos de software.
  • Inspección de código desde el navegador: Los navegadores modernos ofrecen herramientas de desarrollo que permiten inspeccionar el código HTML, CSS y JavaScript de una página web, realizar pruebas y depurar problemas en tiempo real.

Tema 2: El Diseño De Una Página Web

I. Tipos de páginas Web

  • Estáticas: Páginas web cuyo contenido no cambia a menos que se modifique el código fuente. Son simples y no requieren interacción del usuario para actualizarse.
  • Dinamicas: Páginas web cuyo contenido puede cambiar en función de las interacciones del usuario o de datos externos. Utilizan tecnologías del lado del servidor para generar contenido dinámico.
  • Web Apps: Aplicaciones web que funcionan de manera similar a aplicaciones de escritorio o móviles, proporcionando una experiencia de usuario rica y altamente interactiva.
  • Blog: Tipo de página web que se actualiza regularmente con contenido nuevo, generalmente en formato de entradas o artículos. Focalizado en la publicación de noticias, opiniones y reflexiones.

II. Puntos básicos en la elaboración de una página Web

  • Definición del propósito: Determinar el objetivo principal de la página web, ya sea proporcionar información, vender productos o servicios, o permitir la interacción.
  • Planificación de contenido: Organizar y estructurar el contenido que se presentará en la página web para asegurar que sea relevante y accesible para el público objetivo.
  • Diseño: Crear un diseño visual que sea atractivo, funcional y que se alinee con el propósito de la página. Incluye la elección de colores, tipografías y la disposición de elementos.
  • Desarrollo: Codificar la página web utilizando lenguajes y tecnologías apropiadas para construir la estructura y funcionalidad de la página.
  • Pruebas y revisión: Probar la página web en diferentes navegadores y dispositivos para asegurar que funcione correctamente y sea accesible.

    Tema 3: Estructura De Una Página Web

    I. Jerarquía de directorio de un proyecto web

    La jerarquía de directorio organiza los archivos y carpetas de un proyecto web de manera estructurada para facilitar su gestión y mantenimiento.

    II. Elementos de estructura para el diseño de una página Web: Concepto

    1. Regla utilizada para validar y certificar una página web

    Las páginas web se validan utilizando herramientas como el validador de W3C para asegurar que cumplen con los estándares web.

    2. Reglas para escribir HTML

    Algunas reglas incluyen el uso correcto de etiquetas, la anidación adecuada y la inclusión de atributos necesarios.

    3. Estructura básica de un archivo web

    Un archivo HTML básico incluye las etiquetas <!DOCTYPE>, <html>, <head>, <title>, <body>, entre otras.

    4. Etiquetas nombre al menos 40

    Algunas etiquetas HTML incluyen <div>, <span>, <a>, <p>, <h1>, <img>, <ul>, <li>, <table>, <tr>, <td>, <th>, <form>, <input>, <button>, <select>, <option>, <textarea>, <label>, <nav>, <header>, <footer>, <section>, <article>, <aside>, <main>, <figure>, <figcaption>, <audio>, <video>, <source>, <canvas>, <svg>, <iframe>, <meta>, <link>, <style>, <script>.

    5. Atributos

    Los atributos proporcionan información adicional sobre los elementos HTML, como id, class, src, href, alt, title, style, data-*.

    6. Elemento HTML

    Un elemento HTML consiste en una etiqueta de apertura, contenido y una etiqueta de cierre, por ejemplo, <p>Este es un párrafo</p>.

    7. Elementos en línea y en bloque

    Los elementos en línea no inician una nueva línea y ocupan solo el espacio necesario, como <span> y <a>.
